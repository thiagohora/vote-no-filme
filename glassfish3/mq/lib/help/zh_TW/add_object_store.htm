新增物件倉庫

使用此對話方塊，將現有物件倉庫加入管理工具的導航窗格中。加入了物件倉庫之後，您可以連接它，並開始將目標物件和連線工廠物件加入其中。

物件倉庫 中包含您建立的管理物件：目標物件和連線工廠物件。可透過用戶端應用程式呼叫的標準 JNDI 查找碼存取其內容。物件倉庫能以檔案為基礎，還可以是 LDAP 伺服器。如果您要使用以檔案為基礎的倉庫，您需要為其建立資料夾。如果您要使用 LDAP 伺服器，則您必須配置並啟動此伺服器。

物件倉庫標籤：指定物件倉庫的名稱。此名稱將顯示在管理主控台的導航窗格中。

JNDI 命名服務屬性：使用此區域中的控制項，為該物件倉庫定義 JNDI 屬性清單。使用 [名稱] 下拉式清單選取屬性名稱。選取屬性名稱之後，使用 [值] 欄位輸入屬性值。

請參閱 http://java.sun.com/products/jndi/1.2/javadoc/，以取得有關 JNDI 應用程式設計介面的資訊。這需要您設定 java.naming.factory.initial 和 java.naming.provider.url 屬性。

初始環境屬性 (java.naming.factory.initial) 可以有兩種值：

  • 檔案系統倉庫：com.sun.jndi.fscontext.RefFSContextFactory
  • LDAP 伺服器：com.sun.jndi.ldap.LdapCtxFactory

而且還需要您指定認證資訊。但如果您擔心安全問題，則可等到連接時由系統提示您指定此類資訊。

  • 指定屬性名稱和值之後，請按一下 [加入] 按鈕，將屬性加入表中。
  • 若要刪除您已經加入的屬性，請選取此表中的屬性，並按一下 [刪除] 按鈕。
  • 使用 [清除] 按鈕清除全部項目。
  • 若要重新配置屬性，請選取項目，變更其值，使用 [變更] 按鈕重新配置該屬性。
  • 使用 [確定] 按鈕，確定全部屬性設定，並加入物件倉庫。

若要變更現有物件倉庫的屬性，您必須進行如下操作：

  1. 如果尚未取消連接，請從物件倉庫中取消連接。　如果在導航窗格中物件倉庫的圖示看起來像：
    ，則表示取消了物件倉庫的連接。若要取消物件倉庫的連接，請在導航窗格中選取此倉庫，並選擇 [動作] > [取消與物件倉庫的連接]。
  2. 選擇 [動作] > [屬性]，以顯示 [物件倉庫屬性] 對話方塊。

若要將目標物件加入物件倉庫中，請在 Destinations 資料夾上按一下，選擇 [動作] > [新增目標物件]。

若要將連線工廠物件加入物件倉庫中，請在 Connection Factory 資料夾上按一下，選擇 [動作] > [新增連線工廠物件]。
