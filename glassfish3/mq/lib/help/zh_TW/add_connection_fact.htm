新增連線工廠物件

使用此對話方塊配置連線工廠物件，並將其加入物件倉庫。

連線工廠 是 JMS 用戶端用來連接代理程式的受管理物件。此物件也用於指定連接的運作方式，以及用戶端的執行階段如何使用它來存取代理程式。

查找名稱 使用此欄位指定連線工廠物件的 JNDI 查找名稱。

Factory 類型 選擇功能表項目，以指定 Factory 產生的連接類型：ConnectionFactory、QueueConnectionFactory、TopicConnectionFactory。如果訊息必須是分配的事務處理的一部分，則選擇 [XAConnectionFactory]、[XAQueueConnectionFactory] 或 [XATopicConnectionFactory]。

唯讀 按一下此核取方塊使 Factory 連接物件為唯讀。如果物件為唯讀，則用戶端應用程式將無法透過 API 呼叫修改連線工廠屬性。

透過配置連線工廠物件，您可以指定 Factory 產生的全部連接的共有屬性。使用對話方塊中所顯示的標籤來配置連線工廠。請按下列連結以取得該特定標籤的資訊。

  • 按一下 [重設為預設值] 按鈕，將所有對話方塊標籤中的全部欄位重設為其預設值。
  • 按一下 [確定]，建立連線工廠物件。

若要變更連線工廠物件的屬性：

  1. 在管理主控台的導航窗格中選取 Connection Factories 資料夾。
  2. 在管理主控台的結果窗格中選取連線工廠物件。
  3. 選擇 [動作] > [屬性]，顯示 [連線工廠物件屬性] 對話方塊。
