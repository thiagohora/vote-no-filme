QueueBrowsers und ServerSessions

Mit Hilfe der Optionen auf dieser Registerkarte können Sie das Durchsuchen des Warteschlangeninhalts für die Client-Runtime neu konfigurieren und das Verhalten von Sitzungen neu festlegen, die auf einem Anwendungsserver ausgeführt werden.

Obergrenze Warteschlangen-Browser-Abfrage (Anzahl Meldungen): Legt fest, wie viele Meldungen die Client-Runtime beim Durchsuchen des Inhalts eines Warteschlangenziels maximal auf einmal abruft. Standardeinstellung: 1000

Zeitüberschreitung bei Warteschlangen-Browser-Abfrage (Millisekunden): Legt fest, wie lange die Client-Runtime beim Durchsuchen des Inhalts eines Warteschlangenziels maximal auf das Abrufen der Meldungen wartet, bevor sie einen Ausnahmefehler meldet. Standardeinstellung: 60 000 Millisekunden.

Meldungen bis zu Höchstzahl in ServerSessions laden: Legt fest, dass ein Message Queue ConnectionConsumer so viele Meldungen in eine ServerSession-Sitzung laden soll, wie unter maxMessages vorgegeben ist. Lassen Sie diese Option deaktiviert, wenn der Message Queue ConnectionConsumer immer nur eine Meldung auf einmal laden soll. Weitere Informationen dazu finden Sie im Abschnitt Application Server Facilities der JMS-Spezifikation.
