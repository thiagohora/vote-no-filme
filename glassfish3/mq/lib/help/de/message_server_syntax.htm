Syntax der Meldungsserveradressen

Die Syntax für die Angabe einer Meldungsserveradresse lautet wie folgt:

schema://address_syntax

schema und address_syntax werden in folgender Tabelle beschrieben.

Schema Verbindungsdienst Beschreibung Syntax
mq jms
und
ssljms 		Der Message Queue-Portmapper an dem angegebenen Host und Anschluss verarbeitet die Verbindungsanforderung, wobei er eine dynamische Zuweisung eines Anschlusses basierend auf den angegebenen Verbindungsdiensten vornimmt. Sobald der Anschluss bekannt ist, stellt Message Queue die Verbindung her. [hostname][:anschluss][/dienstname]
Standardwerte (nur für JMS-Dienst):
hostname = localhost
anschluss = 7676
dienstname = jms
mqtcp jms Message Queue stellt eine direkte TCP-Verbindung zum angegebenen Host und Anschluss her, um eine Verbindung aufzubauen. hostname:anschluss/jms
mqssl ssljms Message Queue stellt eine direkte, sichere SSL-Verbindung zum angegebenen Host und Anschluss her, um eine Verbindung aufzubauen. hostname:anschluss/ssljms
htttp httpjms Message Queue stellt eine direkte HTTP-Verbindung zum angegebenen Message Queue-Tunnel-Servlet-URL her. (Der Broker muss für den Zugriff auf das Tunnel-Servlet konfiguriert sein.) HTTP-tunnel-servlet-URL
htttps httpsjms Message Queue stellt eine direkte HTTP-Verbindung zum angegebenen Message Queue-Tunnel-Servlet-URL her. (Der Broker muss für den Zugriff auf das Tunnel-Servlet konfiguriert sein.) HTTPS-tunnel-servlet-URL
