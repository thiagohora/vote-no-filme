Zielobjekt hinzufügen

Mit Hilfe dieses Dialogfelds können Sie Objektspeichern Zielobjekte hinzufügen.

Ein Zielobjekt ist eine vom Provider unabhängige Darstellung eines physischen Ziels (Warteschlange oder Thema) in einem Broker.

Suchname Legen Sie in diesem Feld den JNDI-Suchnamen des verwalteten Objekts fest. Dieser Name kann mit dem Zielnamen identisch sein. Einzige Bedingung dabei ist, dass er im Kontext des Objektspeichers eindeutig ist.

Zieltyp Legen Sie den Zieltyp fest. Folgende Optionen stehen zur Wahl: Warteschlange und Thema.

Schreibgeschützt Aktivieren Sie diese Option, wenn das Objekt schreibgeschützt sein soll. Bei schreibgeschützten Objekten können Client-Anwendungen die Attribute von verwalteten Objekten nicht ändern.

Zielname: Legen Sie in diesem Feld den Namen des physischen Ziels auf dem Broker fest, dem das verwaltete Objekt entspricht. Wenn Sie keinen Namen festlegen, wird der Standardname Untitled_Destination_Object verwendet.

Zielbeschreibung: Geben Sie in diesem Feld eine aussagekräftige Beschreibung für das Zielobjekt ein.

  • Wenn alle Felder auf ihre Standardwerte zurückgesetzt und Ihre Änderungen damit verworfen werden sollen, klicken Sie auf Zurücksetzen auf Standardwerte.
  • Wenn das Zielobjekt mit den von Ihnen eingegebenen Informationen erstellt werden soll, klicken Sie auf OK.

So ändern Sie die Eigenschaften eines Zielobjekts:

  1. Wählen Sie im Navigationsbereich der Administrationskonsole den Ordner Ziele aus.
  2. Wählen Sie im Ergebnisbereich der Administrationskonsole das Zielobjekt aus.
  3. Wählen Sie den Befehl Aktionen > Eigenschaften, um das Dialogfeld Eigenschaften Zielobjekt zu öffnen.
