Objektspeicher hinzufügen

Mit Hilfe dieses Dialogfelds können Sie dem Navigationsbereich Ihrer Administratorkonsole einen bereits vorhandenen Objektspeicher hinzufügen. Nach dem Hinzufügen des Objektspeichers können Sie eine Verbindung zu diesem Objektspeicher herstellen und damit fortfahren, ihm Zielobjekte und Verbindungs-Factory-Objekte hinzuzufügen.

Ein Objektspeicher enthält die von Ihnen erstellten Verwaltungsobjekte: Zielobjekte und Verbindungs-Factory-Objekte. Der Zugriff auf deren Inhalt erfolgt über Standard-JNDI-Abfragen, die von der Client-Anwendung aufgerufen werden. Objektspeicher können entweder dateibasiert sein oder es kann sich bei ihnen um einen LDAP-Server handeln. Wenn Sie einen dateibasierten Objektspeicher verwenden möchten, müssen Sie einen Ordner dafür erstellen. Soll ein LDAP-Server verwendet werden, müssen Sie den Server konfigurieren und starten.

Objektspeicherbeschriftung: Legt den Namen des Objektspeichers fest. Unter diesem Namen wird der Objektspeicher im Navigationsbereich der Administratorkonsole angezeigt.

Eigenschaften JNDI-Benennungsdienst: Mit den Optionen in diesem Bereich können Sie die Liste der JNDI-Eigenschaften für diesen Objektspeicher definieren. Wählen Sie in der Dropdown-Liste Name einen Eigenschaftsnamen aus. Geben Sie anschließend im Feld Wert den Eigenschaftswert ein.

Informationen zur JNDI-API finden Sie unter http://java.sun.com/products/jndi/1.2/javadoc/. Sie müssen die Eigenschaften java.naming.factory.initial und java.naming.provider.url festlegen.

Die anfängliche Kontexteigenschaft (java.naming.factory.initial) kann zwei Werte aufweisen:

  • Dateisystemspeicher: com.sun.jndi.fscontext.RefFSContextFactory
  • LDAP-Server: com.sun.jndi.ldap.LdapCtxFactory

Darüber hinaus müssen Sie die Authentifizierungsinformationen eingeben. Wenn Ihnen das nicht sicher genug ist, sollten Sie mit der Eingabe dieser Informationen warten, bis Sie beim Verbindungsaufbau dazu aufgefordert werden.

  • Klicken Sie nach dem Eingeben des Namens und des Wertes für die Eigenschaft auf Hinzufügen, um der Tabelle die Eigenschaft hinzuzufügen.
  • Wenn Sie eine bereits hinzugefügte Eigenschaft wieder löschen möchten, wählen Sie die entsprechende Eigenschaft in der Tabelle aus und klicken Sie dann auf Löschen.
  • Wenn alle Einträge gelöscht werden sollen, klicken Sie auf Alle löschen.
  • Wenn Sie eine Eigenschaft ändern möchten, wählen Sie den entsprechenden Eintrag aus, ändern Sie den Wert der Eigenschaft und klicken Sie dann auf Ändern.
  • Klicken Sie auf OK, um alle Einstellungen zu bestätigen und den Objektspeicher hinzuzufügen.

So ändern Sie die Eigenschaften eines vorhandenen Objektspeichers:

  1. Trennen Sie, wenn nicht bereits geschehen, die Verbindung zum Objektspeicher. Wenn die Verbindung getrennt ist, sieht das Objektspeicher-Symbol im Navigationsbereich wie folgt aus:
    Wenn Sie die Verbindung zu einem Objektspeicher trennen möchten, wählen Sie im Navigationsbereich den Objektspeicher aus, und klicken Sie auf Aktionen > Verbindung mit Objektspeicher trennen.
  2. Wählen Sie den Befehl Aktionen > Eigenschaften, um das Dialogfeld Objektspeichereigenschaften zu öffnen.

Wenn Sie dem Objektspeicher ein Zielobjekt hinzufügen möchten, klicken Sie auf den Ordner Ziele und wählen Sie den Befehl Aktionen > Zielobjekt hinzufügen.

Wenn Sie dem Objektspeicher ein Verbindungs-Factory-Objekt hinzufügen möchten, klicken Sie auf den Ordner Verbindungs-Factory und wählen Sie den Befehl Aktionen > Verbindungs-Factory-Objekt hinzufügen.
