QueueBrowsers y ServerSessions

Use esta ficha para reconfigurar la exploración de la cola para el motor de ejecución cliente, así como para volver a especificar el comportamiento de las sesiones que se ejecutan en un servidor de aplicaciones.

Tamaño de recuperación del explorador de la cola Especifica el máximo número de mensajes que el motor de ejecución cliente recuperará de una vez al examinar el contenido de un destino de cola. Por defecto: 1000

Tiempo de espera de recuperación del explorador de la cola (miliseg.) Especifica el tiempo máximo que el motor de ejecución cliente esperará para recuperar mensajes al examinar el contenido de un destino de cola antes de iniciar una excepción. Por defecto: 60000 milisegundos.

Cargar hasta el máximo de mensajes en ServerSessions Especifica que un objeto ConnectionConsumer de Message Queue debe cargar hasta el número maxMessages de mensajes en una sesión de ServerSession. Deje la casilla sin activar para indicar que un objeto ConnectionConsumer de Message Queue debe cargar un sólo mensaje a la vez. Vea la sección "Application Server Facilities" de la especificación JMS para más información.
