Gestión de conexiones

Use la ficha "Gestión de conexiones" para configurar las conexiones que se crearán mediante esta fábrica.

Lista de direcciones del servidor de mensajes Especifica una lista de direcciones del servidor de mensajes (una o varias), separadas por comas, cada una de las cuales corresponde a una instancia diferente del agente a la que se puede conectar el motor de ejecución cliente. Cada una de las direcciones de la lista incluye el nombre y el puerto del host y el servicio de conexión para efectuar la conexión.
Para obtener más información acerca de la sintaxis de las direcciones del servidor de mensajes, haga clic aquí

Orden de la lista de direcciones Especifica si los intentos para establecer una conexión se realizan en el orden en que aparecen las direcciones en la lista del servidor de mensajes (PRIORIDAD) o en un orden aleatorio (ALEATORIO). Si varios clientes intentan conectarse usando la misma fábrica de conexión, puede establecer un orden aleatorio para evitar que se conecten todos a la misma dirección.

Número de intentos en la lista de direcciones Especifica el número de veces que el motor de ejecución cliente intentará establecer (o restablecer) una conexión en la lista de direcciones del servidor. El valor -1 indica que el número de intentos es ilimitado.

Activar reconexión con el servicio de mensajes Si su valor es verdadero, el motor de ejecución cliente podrá intentar reconectarse a un servidor de mensajes (o a la lista de direcciones, en el caso de una lista de servidor de mensajes) cuando se pierda una conexión.

Número de intentos de reconexión Especifica el número de intentos de reconexión para cada dirección de la lista de direcciones del servidor de mensajes antes de que el motor de ejecución cliente avance a la siguiente dirección de la lista. El valor -1 indica que el número de intentos es ilimitado.

Retraso de reconexión con el servicio de mensajes (miliseg.) Especifica el retraso (en milisegundos) entre los intentos de reconexión para cada dirección de la lista de direcciones del servidor de mensajes antes de que el motor de ejecución cliente avance a la siguiente dirección de la lista. Si el intervalo es demasiado corto, el agente no podrá recuperarse. Si es demasiado largo, la reconexión puede suponer un retraso inaceptable.

Intervalo de pulso de sondeo de conexión (segundos) Especifica el intervalo mínimo (en segundos) en el que el motor de ejecución cliente comprobará periódicamente que su conexión con el agente es buena.
