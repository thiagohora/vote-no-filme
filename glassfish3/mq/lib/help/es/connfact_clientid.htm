Identificación de cliente

Utilice esta ficha para volver a especificar los valores predeterminados del nombre de usuario y la contraseña, así como para indicar de nuevo la forma en que se identificará un cliente (ClientID) ante un agente. Los valores de ClientID se usan principalmente para llevar un seguimiento de las suscripciones duraderas.

Nombre de usuario predeterminado Especifica el nombre de usuario que se utilizará de forma predeterminada para autenticarse ante el agente. El valor predeterminado es guest.

Contraseña predeterminada Especifica la contraseña que se utilizará de forma predeterminada para autenticarse ante el agente. El valor predeterminado es guest.

Desactivar API JMS para setClientID() Especifica que la aplicación cliente no puede usar llamadas a API para cambiar el valor ClientID. Esto supone una ventaja para el administrador, ya que le permite controlar la asignación de los ClientID.

Si activa esta casilla, puede usar el campo Configurar ClientID como para especificar una identificación de cliente. Si no lo especifica, el sistema asignará un valor predeterminado compuesto por la dirección IP del host del cliente. Si bien ello se hace para comodidad de los programadores, impide que una aplicación cliente pueda crear más de una conexión: el sistema no permite dos conexiones con un mismo valor de ClientID.

Configurar ClientID como Especifica un identificador de cliente. Si no especifica este valor y deshabilitó la llamada a API que define el Id. de cliente, el sistema asignará un valor predeterminado con la dirección IP del host del cliente. Si bien ello se hace para comodidad de los programadores, impide que una aplicación cliente pueda crear más de una conexión: el sistema no permite dos conexiones con un mismo valor de ClientID.
