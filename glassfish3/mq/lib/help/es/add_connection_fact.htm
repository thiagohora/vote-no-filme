Agregar objeto de fábrica de conexión

Use este diálogo para configurar un objeto de fábrica de conexión y agregarlo a un almacén de objetos.

Las fábricas de conexión son objetos administrados que utilizan los clientes JMS para conectar con el agente. Este objeto se usa también para especificar el comportamiento de la conexión y el tipo de acceso al agente del motor de ejecución cliente que lo utiliza.

Nombre de búsqueda Use este campo para especificar el nombre de búsqueda JNDI del objeto de fábrica de conexión.

Tipo de fábrica Elija el elemento de menú que especifica el tipo de conexiones que produce la fábrica: ConnectionFactory, QueueConnectionFactory o TopicConnectionFactory. Si el mensaje debe formar parte de una transacción distribuida, elija XAConnectionFactory, XAQueueConnectionFactory o XATopicConnectionFactory.

Sólo lectura Haga clic en esta casilla de verificación para que el objeto de fábrica de conexión sea de sólo lectura. Si el objeto es de sólo lectura, las aplicaciones cliente no podrán modificar sus atributos mediante llamadas de API.

Al configurar un objeto de fábrica de conexión se especifican las propiedades comunes a todas las conexiones que produce la fábrica. Utilice las fichas mostradas en el diálogo para configurar la fábrica de conexión. Haga clic en uno de los enlaces siguientes para más información sobre una ficha determinada.

  • Haga clic en el botón Usar valores predeterminados para usar el valor predeterminado en todos los campos de todas las fichas del diálogo.
  • Haga clic en Aceptar para crear el objeto de fábrica de conexión.

Para cambiar las propiedades de un objeto de fábrica de conexión:

  1. Seleccione la carpeta Fábricas de conexión en el panel de navegación de la consola de administración.
  2. Seleccione el objeto de fábrica de conexión en el panel de resultados de la consola de administración.
  3. Elija Acciones > Propiedades para abrir el diálogo Propiedades del objeto de fábrica de conexión.
