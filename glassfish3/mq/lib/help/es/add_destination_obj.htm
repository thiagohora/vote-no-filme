Agregar objeto de destino

Use este diálogo para agregar un objeto de destino a un almacén de objetos.

Un objeto administrado de destino es una representación independiente del proveedor de un destino físico (una cola o un tema) en un agente.

Nombre de búsqueda Use este campo para especificar el nombre de búsqueda JNDI del objeto administrado. Este nombre puede ser el mismo que el nombre del destino; la única restricción es que debe ser único en el contexto del almacén de objetos.

Tipo de destino Haga clic en el botón de radio que corresponde al tipo de destino: cola o tema.

Sólo lectura Haga clic en esta casilla de verificación para que el objeto sea de sólo lectura. Si el objeto es de sólo lectura, las aplicaciones cliente no podrán modificar los atributos de objetos administrados mediante llamadas de API.

Nombre del destino Use este campo para especificar el nombre del destino físico en el agente al que corresponde este objeto administrado. Si no lo especifica, se usará el nombre predeterminado de Untitled_Destination_Object.

Descripción del destino Use este campo para indicar una descripción útil de este objeto de destino.

  • Haga clic en el botón Usar valores predeterminados para usar el valor predeterminado en todos los campos.
  • Haga clic en Aceptar para crear el objeto de destino.

Para cambiar las propiedades de un objeto de destino:

  1. Seleccione la carpeta Destinos en el panel de navegación de la consola de administración.
  2. Seleccione el objeto de destino en el panel de resultados de la consola de administración.
  3. Elija Acciones > Propiedades para abrir el diálogo Propiedades del objeto de destino.
