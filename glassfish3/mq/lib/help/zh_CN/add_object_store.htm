添加对象存储

使用此对话框，可以将现有对象存储添加到管理工具的导航窗格中。添加对象存储后，可与之连接并继续为其添加目的地对象和连接工厂对象。

对象存储包含用户创建的管理对象：目的地对象和连接工厂对象。可以通过客户机应用程序调用标准的 JNDI 查找代码访问其中的内容。对象存储可以基于文件，也可以是 LDAP 服务器。如果要使用基于文件的存储，则需要为其创建文件夹。如果要使用 LDAP 服务器，则必须配置并启动服务器。

对象存储标签：指定对象存储库的名称。此名称将显示在管理控制台的导航窗格中。

JNDI 命名服务属性：使用此区域中的控件，可以为对象存储定义 JNDI 属性列表。使用“名称”下拉列表选择属性名。选择属性名后，使用“值”字段输入属性值。

有关 JNDI 应用程序编程接口的信息，请访问 http://java.sun.com/products/jndi/1.2/javadoc/。必须设置 java.naming.factory.initial 和 java.naming.provider.url 属性。

初始上下文属性 (java.naming.factory.initial) 可以具有以下两个值：

  • 文件系统存储：com.sun.jndi.fscontext.RefFSContextFactory
  • LDAP 服务器：com.sun.jndi.ldap.LdapCtxFactory

还须指定验证信息。但是，如果安全性对您来说很重要，则可以在连接时等待系统提示您输入此信息。

  • 指定属性名和属性值后，单击“添加”按钮将属性添加到表中。
  • 要删除已经添加的属性，请从列表中选择属性并单击“删除”按钮。
  • 使用“清除”按钮可以清除所有项。
  • 要重新配置属性，请选择相应的项并更改其值，然后使用“更改”按钮重新配置此属性。
  • 使用“确定”按钮提交所有属性设置并添加对象存储。

要更改现有对象存储的属性，必须执行下列操作：

  1. 与对象存储断开连接（如果尚未断开）。当导航窗格中的对象存储图标呈现以下外观时，表示对象存储已断开：
    要断开对象存储库，请选择导航窗格中的存储库并选择“操作”>“断开对象存储库”。
  2. 选择“操作”>“属性”显示“对象存储库属性”对话框。

要将目的地对象添加到对象存储库中，请单击“目的地”文件夹，然后选择“操作”>“添加目的地对象”。

要将连接工厂添加到对象存储中，请单击“连接工厂”文件夹，然后选择“操作”>“添加连接工厂对象”。
