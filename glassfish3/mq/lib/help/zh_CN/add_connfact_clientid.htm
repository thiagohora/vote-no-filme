客户身份验证

使用此选项卡，可以指定默认的用户名和密码，还可以指定客户机对于代理的标识 (ClientID) 方式。ClientId 值主要用于追踪长期用户。

默认用户名 指定将用于通过代理进行验证的默认用户名。默认值为 guest。

默认密码 指定将用于通过代理进行验证的默认密码。默认值为 guest。

禁用 setClientID() JMS API 指定阻止客户机应用程序使用 API 调用更改 ClientID。一般说来，您作为管理员最好控制 ClientID 的分配。

如果选中此复选框，则可以使用“将 ClientID 配置为”字段为客户机指定 ClientID。如果不指定这样的值，系统将分配由客户机的主机 IP 地址组成的默认值。尽管使用默认值为开发人员提供了方便，但同时也使客户机应用程序无法创建多个连接：系统不允许两个连接具有相同的 ClientID。

将 ClientID 配置为 指定客户机的标识符。如果用户未指定该值并禁用了设置 ClientID 的 API 调用，则系统将指定由客户机主机的 IP 地址组成的默认值。尽管使用默认值为开发人员提供了方便，但同时也使客户机应用程序无法创建多个连接：系统不允许两个连接具有相同的 ClientID。
