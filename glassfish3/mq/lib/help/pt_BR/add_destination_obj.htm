Adicionar objeto de destino

Utilize esta caixa de diálogo para adicionar um objeto de destino a um armazenamento de objetos.

Um objeto de destino administrado é uma representação de um destino físico (uma fila ou um tópico) que não depende de fornecedor em um broker.

Nome de pesquisa Utilize este campo para especificar o nome de pesquisa JNDI do objeto administrado. Este nome pode ser igual ao nome de destino. A única restrição é que deve ser exclusivo no contexto do armazenamento de objetos.

Tipo do destino Clique no botão de opção que corresponde ao tipo do destino: fila ou tópico.

Somente leitura Clique nesta caixa de seleção para tornar o objeto somente leitura. Se o objeto for somente leitura, os aplicativos clientes não poderão modificar os atributos do objeto administrado através de chamadas API.

Nome do destino Utilize este campo para especificar o nome do destino físico no broker ao qual corresponde o objeto administrado. Se você não especificar um nome, o nome padrão Untitled_Destination_Object será utilizado.

Descrição do destino Utilize este campo para introduzir uma descrição útil deste objeto de destino.

  • Clique no botão Restaurar para padrões para restaurar todos os campos com seus valores padrão.
  • Clique em OK para criar o objeto de destino.

Para alterar as propriedades de um objeto de destino

  1. Selecione a pasta Destinos no painel de navegação do console de administração.
  2. Selecione o objeto de destino no painel de resultados do console de administração.
  3. Escolha Ações > Propriedades para abrir a caixa de diálogo Propriedades do objeto de destino.
