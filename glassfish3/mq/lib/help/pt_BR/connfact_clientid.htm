Identificação do cliente

Utilize esta aba para especificar novamente os valores padrão do nome de usuário e senha, e para especificar novamente como um cliente será identificado (ClientID) para um broker. Os valores do ClientId são utilizados principalmente para controlar os assinantes permanentes.

Nome de usuário padrão Especifica o nome de usuário padrão que será utilizado para autenticar no broker. O valor padrão é guest.

Senha padrão Especifica a senha padrão que será utilizada para autenticar no broker. O valor padrão é guest.

Desativar a API JMS setClientID() Especifica que o aplicativo cliente é impedido de utilizar chamadas API para alterar o ClientID. Geralmente é melhor que você, como administrador, controle a atribuição de ClientIDs.

Se marcar esta caixa, poderá utilizar o campo Configurar ClientID como para especificar um ClientID para o cliente. Se tal valor não for especificado, o sistema se encarrega de atribuir um valor padrão que corresponde ao endereço IP do host do cliente. Embora seja útil para os desenvolvedores, esta ação impede que o aplicativo cliente crie mais de uma conexão: o sistema não permite duas conexões com o mesmo ClientID.

Configurar ClientID como Especifica um identificador para o cliente. Se este valor não for especificado e a chamada API que define o ClientID estiver desativada, o sistema se encarrega de atribuir um valor padrão que corresponde ao endereço IP do host do cliente. Embora seja útil para os desenvolvedores, esta ação impede que o aplicativo cliente crie mais de uma conexão: o sistema não permite duas conexões com o mesmo ClientID.
