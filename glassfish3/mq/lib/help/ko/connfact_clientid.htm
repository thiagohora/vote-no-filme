클라이언트 아이디

이 탭을 사용하여 사용자 이름과 비밀번호의 기본값을 지정하고 브로커에서 클라이언트를 식별(ClientID)하는 방법을 지정합니다. ClientID 값은 주로 영구 가입자의 추적에 사용됩니다.

기본 사용자 이름 브로커에서 인증에 사용할 기본 사용자 이름을 지정합니다. 기본값은 guest입니다.

기본 비밀번호 브로커에서 인증에 사용할 기본 비밀번호를 지정합니다. 기본값은 guest입니다.

setClientID() JMS API 비활성화 클라이언트 응용 프로그램이 ClientID 변경을 위해 API 호출을 사용할 수 없도록 지정합니다. 일반적으로 관리자가 ClientID 지정을 제어하는 것이 바람직합니다.

이 확인란을 선택하면 ClientID를 다른 이름으로 구성 필드를 사용하여 해당 클라이언트의 ClientID를 지정할 수 있습니다. 값을 지정하지 않으면 시스템은 클라이언트의 호스트 IP 주소로 구성된 기본값을 지정합니다. 개발자의 편의를 위한 것이지만, 클라이언트 응용 프로그램은 둘 이상의 연결을 생성할 수 없습니다. 시스템은 2개의 연결이 동일한 ClientID를 사용하도록 허용하지 않습니다.
