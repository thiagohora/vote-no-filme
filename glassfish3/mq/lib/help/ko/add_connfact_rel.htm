안정성 및 흐름 제어

이 탭을 사용하여 클라이언트 런타임에 의한 Message Queue 제어 메시지(브로커 확인) 및 흐름 제어 조치의 사용을 지정합니다.

생성된 메시지의 메시지 서비스 확인 생성자 클라이언트로부터의 메시지에 대한 브로커 확인을 지정합니다.

  • true로 설정된 경우 브로커는 생성자 클라이언트로부터의 모든 JMS 메시지(지속성 및 비지속성) 수신을 확인하며, 생성자 클라이언트 스레드는 이 확인의 대기를 차단합니다.
  • false로 설정되면 브로커는 생성자 클라이언트로부터의 어떤 JMS 메시지(지속성 및 비지속성) 수신도 확인하지 않으며, 생성자 클라이언트 스레드는 브로커 확인의 대기를 차단하지 않습니다.
  • 값을 지정하지 않으면 브로커는 지속성 메시지의 수신만을 확인하고 생성자 클라이언트 스레드는 브로커의 확인 대기를 차단합니다.

클라이언트 확인의 메시지 서비스 확인 사용자 클라이언트 확인에 대한 브로커 확인을 지정합니다.

  • true로 설정된 경우 브로커는 모든 사용자 클라이언트 확인을 확인하며, 사용자 클라이언트 스레드는 그러한 브로커 확인의 대기를 차단합니다.
  • false로 설정된 경우 브로커는 어떤 사용자 클라이언트 확인도 확인하지 않으며, 사용자 클라이언트 스레드는 그러한 브로커 확인의 대기를 차단하지 않습니다.
  • 이 옵션을 지정하지 않은 경우, 브로커는 AUTO_ACKNOWLEDGE 및 CLIENT_ACKNOWLEDGE 모드에 대해 사용자 클라이언트 확인을 확인하지만(그리고 사용자 클라이언트 스레드는 그러한 브로커 확인의 대기를 차단합니다), DUPS_OK_ACKNOWLEDGE 모드에 대해서는 사용자 클라이언트 확인을 확인하지 않습니다(그리고 사용자 클라이언트 스레드는 차단하지 않습니다).

메시지 서비스 확인 시간 초과(밀리초) 클라이언트 런타임에서 브로커의 확인을 기다리는 최대 시간(밀리초)을 지정합니다. 이 시간이 초과되면 예외가 발생합니다. 기본값인 0은 시간 초과가 없음을 의미합니다. 즉 클라이언트 런타임 대기 시간의 제한이 없습니다.

연결 흐름 수 측정된 일괄 처리에서 JMS 메시지의 최대 수를 지정합니다. 이 JMS 메시지 수가 클라이언트 런타임에 전달되면 전달이 일시적으로 중단되어 보류된 제어 메시지를 모두 전달할 수 있도록 합니다. 클라이언트 런타임의 알림 메시지를 받으면 JMS 메시지 전달이 다시 시작되어 지정된 수에 이를 때까지 계속됩니다. 지정된 수가 0이면 측정된 일괄 처리에서 JMS 메시지의 수에 제한이 없습니다. 0이 아닌 값으로 설정하면 클라이언트 런타임에서 메시지 흐름을 측정하므로 Message Queue 제어 메시지는 JMS 메시지 전달 폭주로 인해 차단되지 않고 측정된 일괄 처리 사이의 클라이언트로 전달될 수 있습니다.

연결 흐름 제한 사용 (클라이언트 런타임에 전달되었지만 사용되지 않은 메시지 수에 대한) 연결 흐름 제한의 사용 여부를 지정합니다. 이 확인란이 선택되면 연결 흐름 제한 필드에 양수 값을 지정해야 합니다.

연결 흐름 제한(연결당 사용되지 않은 메시지) 연결을 통해 전달되고 클라이언트 런타임에 버퍼링된 후 사용 대기하는 메시지 수의 제한을 지정합니다(연결 흐름 제한 사용이 선택된 경우에만 사용). (연결 흐름 수에서 관리하는 흐름 측정에 따라) 클라이언트로 전달되는 JMS 메시지의 수가 이 제한을 초과하면 메시지 전달이 중단됩니다. 사용되지 않는 메시지의 수가 이 등록 정보의 설정값보다 낮아지는 경우에만 메시지 전달이 다시 시작됩니다. 이 제한은 메시지 처리 시간이 오래 걸리는 사용자 클라이언트에 대기 중인 메시지가 너무 많아 메모리 부족이 발생하는 것을 방지합니다.

사용자 흐름 제한(사용자당 사용되지 않은 메시지) 연결을 통해 전달되고 클라이언트 런타임에 버퍼링될 수 있는 사용자당 메시지 수의 제한을 지정합니다. 이 제한은 두 가지 용도를 갖습니다. 다중 사용자 대기열 전달의 경우 사용자간의 로드 균형 조정을 향상시킬 수 있고(한 사용자에게 보내는 메시지 수의 불균형 방지), 연결된 한 사용자가 다른 연결된 사용자에게 과도하게 메시지를 보내는 것을 방지하는 데 기여합니다. 어떤 사용자에 대해서도 클라이언트 런타임에 전달되는 JMS 메시지의 수가 이 제한을 초과하면 해당 사용자에 대한 메시지 전달이 중단됩니다. 해당 사용자의 사용되지 않은 메시지 수가 사용자 흐름 임계값에 설정된 값보다 낮아지는 경우에만 다시 시작됩니다

연결된 모든 사용자에 대해 버퍼링된 전체 메시지 수가 연결 흐름 제한을 초과하면 해당 연결을 통한 메시지 전달은 그 전체 메시지 수가 제한 값보다 낮아질 때까지 중단됩니다. 사용자 흐름 제한을 지정했다면 가장 특별한 경우를 제외하고는 연결 흐름 제한을 설정할 필요가 없으며, 대부분의 응용 프로그램에서 메시지 흐름 측정에 대한 요구 사항은 연결 흐름 제한 대신 사용자 흐름 제한을 설정하면 해결됩니다.

사용자 흐름 임계값(백분율) 클라이언트 런타임에 버퍼링된 사용자당 메시지의 수를 사용자 흐름 제한에 대한 백분율로 지정하며, 이 값보다 낮으면 해당 사용자에 대한 메시지 전달이 다시 시작됩니다.
