QueueBrowsers e ServerSessions

Utilizzare questa scheda per configurare la navigazione della coda per il runtime del client e per specificare il comportamento di sessioni in esecuzione in un server di applicazioni.

Dimensione di recupero browser della coda Specifica il numero massimo di messaggi che il runtime del client recupererà in una volta, durante la navigazione dell'indice della destinazione per la coda. Predefinito: 1000

Timeout di recupero browser della coda (millisecondi) Specifica il tempo massimo che il runtime del client attenderà per recuperare messaggi, durante la navigazione dell'indice di una destinazione per la coda prima del lancio di una eccezione. Predefinito: 60000 millisecondi.

Carica fino a un numero massimo di messaggi sulle sessioni del server Specifica che un ConnectionConsumer di Message Queue deve caricare fino a maxMessages numero di messaggi in una sessione del ServerSession. Lasciare la casella non selezionata per specificare che un ConnectionConsumer di Message Queue deve caricare soltanto un singolo messaggio alla volta. Per ulteriori informazioni vedere la sezione relativa alle funzionalità del server di applicazioni della specifica JMS.
