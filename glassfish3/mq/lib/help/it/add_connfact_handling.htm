Gestione di·connessione

Utilizzare la scheda Gestione di·connessione per configurare le connessioni che verranno create da questa factory.

Elenco indirizzi server dei messaggi Specifica un elenco di indirizzi del server dei messaggi (uno o più), separati da virgole, ognuno corrispondente a un'istanza del broker diversa alla quale il runtime client può connettersi. Ogni indirizzo nell'elenco specifica il nome host, la porta host e il servizio di connessione per la connessione.
Per ulteriori informazioni sulla sintassi relativa all'indirizzo del server dei messaggi, fare clic qui

Ordine elenco indirizzi Specifica se i tentativi di connessione sono nell'ordine degli indirizzi incluso nell'elenco di indirizzi del server dei messaggi (PRIORITÀ) o in ordine casuale (CASUALE). Se vari client tentano di eseguire la connessione utilizzando la stessa factory di connessione, è consigliabile utilizzare l'ordine casuale per evitare che vengano connessi allo stesso indirizzo.

Numero di iterazioni elenco indirizzi Specifica quante volte il runtime del client eseguirà un'iterazione tramite l'elenco di indirizzi del server dei messaggi nel tentativo di stabilire (o ristabilire) la connessione. Il valore -1 indica che il numero di tentativi è illimitato.

Abilitare la riconnessione automatica al server dei messaggi Il valore True indica che il runtime del client eseguirà un tentativo di riconnessione al server (o all'elenco di indirizzi in un elenco di server di messaggi) quando la connessione viene interrotta.

Numero di tentativi di riconnessione per indirizzo Indica il numero di tentativi di riconnessione per ogni indirizzo dell'elenco degli indirizzi del server di messaggi prima che il runtime del client inizi a provare l'indirizzo successivo nell'elenco. Il valore -1 indica che il numero di tentativi è illimitato.

Intervallo di riconnessione per indirizzo (millisecondi) Specifica l'intervallo in millisecondi tra i tentativi di riconnessione per ogni indirizzo nell'elenco di indirizzi del server di messaggi prima che il runtime del client inizi a provare l'indirizzo successivo nell'elenco. Se l'intervallo di tempo è troppo breve non consente al broker di completare il recupero. Se è troppo lungo, la riconnessione potrebbe richiedere un ritardo inaccettabile.

Intervallo ping connessione (secondi) Indica l'intervallo minimo in secondi con cui il runtime del client verifica periodicamente lo stato della connessione al broker.
