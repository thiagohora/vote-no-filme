Aggiungi archivio degli oggetti

Utilizzare questa finestra di dialogo per aggiungere un archivio degli oggetti esistente al riquadro di navigazione dello strumento di amministrazione. Dopo aver aggiunto l'archivio degli oggetti, è possibile connettersi a tale archivio e aggiungere oggetti di destinazione e oggetti ConnectionFactory.

Un archivio degli oggetti contiene gli oggetti di amministrazione creati: oggetti di destinazione e oggetti ConnectionFactory. Tali contenuti sono accessibili attraverso un codice di ricerca JNDI standard denominato dall'applicazione del client. Un archivio degli oggetti può essere un server LDAP o basato sui file. Se si desidera utilizzare un archivio basato sui file, è necessario creare una cartella. Se si desidera utilizzare un server LDAP, è necessario configurare e avviare tale server.

Etichetta dell'archivio degli oggetti: Specifica il nome dell'archivio degli oggetti. Il nome viene visualizzato nel riquadro di navigazione della console di amministrazione.

Proprietà del servizio di denominazione JNDI: utilizzare i controlli in questa sezione per definire l'elenco delle proprietà JNDI per questo archivio degli oggetti. Utilizzare l'elenco a discesa Nome per selezionare un nome di proprietà. Dopo avere selezionato il nome della proprietà, utilizzare il campo Valore per inserire il valore della proprietà.

Vedere http://java.sun.com/products/jndi/1.2/javadoc/ per informazioni sull'interfaccia di programmazione dell'applicazione JNDI. È necessario impostare le proprietà java.naming.factory.initial e java.naming.provider.url .

La proprietà di contesto iniziale (java.naming.factory.initial) può essere associata a due valori:

  • Archivio file system: com.sun.jndi.fscontext.RefFSContextFactory
  • Server LDAP: com.sun.jndi.ldap.LdapCtxFactory

Inoltre, è necessario specificare le informazioni di autenticazione. Tuttavia, se la sicurezza è una priorità, è possibile attendere la visualizzazione di un messaggio che richiede tali informazioni al momento della connessione.

  • Dopo aver specificato il nome e il valore della proprietà, fare clic sul pulsante Aggiungi per aggiungere la proprietà alla tabella.
  • Per eliminare una proprietà già aggiunta, selezionare una proprietà nella tabella e fare clic sul pulsante Elimina.
  • Utilizzare il pulsante Cancella per eliminare tutte le voci.
  • Per riconfigurare una proprietà, selezionare una voce, modificarne il valore e utilizzare il pulsante Modifica per riconfigurare la proprietà.
  • Utilizzare il pulsante OK per eseguire il commit di tutte le impostazioni della proprietà e aggiungere l'archivio degli oggetti.

Per modificare le proprietà di un archivio degli oggetti esistente, è necessario eseguire la procedura riportata di seguito:

  1. Disconnettersi dall'archivio degli oggetti se il collegamento non è ancora stato interrotto. Un archivio degli oggetti risulta disconnesso se l'icona nel riquadro di navigazione appare così:
    Per eseguire la disconnessione di un archivio degli oggetti, selezionare l'archivio nel riquadro di navigazione e selezionare Azioni > Disconnetti dall'archivio degli oggetti.
  2. Selezionare Azioni > Proprietà per aprire la finestra di dialogo Proprietà dell'archivio degli oggetti.

Per aggiungere un oggetto di destinazione all'archivio degli oggetti, fare clic sulla cartella Destinazioni e selezionare Azioni > Aggiungi oggetto di destinazione.

Per aggiungere una factory di connessione all'archivio degli oggetti, fare clic sulla cartella Factory di connessione e selezionare Azioni > Aggiungi oggetto ConnectionFactory.
