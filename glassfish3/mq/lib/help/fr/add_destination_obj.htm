Ajouter objet de destination

Utilisez cette boîte de dialogue pour ajouter un objet de destination à un magasin d'objets.

Un objet administré par destination est une représentation, indépendante du fournisseur, d'une destination physique (une file d'attente ou une rubrique) pour un courtier.

Nom de recherche Utilisez ce champ pour spécifier le nom de recherche JNDI de l'objet administré. Ce nom peut être le même que le nom de destination ; la seule contrainte est qu'il doit être unique dans le contexte du magasin d'objets.

Type de destination Activez la case d'option correspondant au type de destination : file d'attente ou rubrique.

Lecture seule Cochez cette case pour que l'objet soit en lecture seule. Si l'objet est en lecture seule, les applications clientes ne pourront pas modifier les attributs de l'objet administré à l'aide d'appels api.

Nom destination Utilisez ce champ pour spécifier le nom de la destination physique sur le courtier à laquelle cet objet administré correspond. Si vous ne spécifiez aucun nom, le nom par défaut est Untitled_Destination_Object.

Description de la destination Utilisez ce champ pour saisir une description utile de cet objet destination.

  • Cliquez sur le bouton Restaurer les paramètres par défaut pour réinitialiser tous les champs sur leurs valeurs par défaut.
  • Cliquez sur OK pour créer l'objet de destination.

Pour modifier les propriétés d'un objet de destination

  1. Sélectionnez le dossier Destinations dans la fenêtre de navigation de la console d'administration.
  2. Sélectionnez l'objet de destination dans la fenêtre des résultats de la console d'administration.
  3. Sélectionnez Actions > Propriétés pour afficher la boîte de dialogue Propriétés de l'objet de destination.
