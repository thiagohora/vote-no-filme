QueueBrowsers et ServerSessions

Utilisez cet onglet pour configurer la navigation des files d'attente pour le moteur d'exécution du client et afin de spécifier le comportement des sessions exécutées sur un serveur d'application.

Taille de récupération du navigateur de file d'attente Spécifie le nombre maximal de messages que le moteur d'exécution d'un client peut extraire à la fois, en parcourant le contenu d'une destination de file d'attente. Par défaut : 1000

Temporisation de la récupération du navigateur de la file d'attente (en millièmes de seconde) Spécifie le délai maximal d'attente du moteur d'exécution d'un client en mode extraction, lorsqu'il parcourt le contenu d'une destination de file d'attente, avant qu'il génère une exception. Par défaut : 60000 millisecondes.

Charger le nombre maximum de messages sur ServerSessions : spécifie qu'une connexion consommateur (ConnectionConsumer) Message Queue peut charger le nombre maximal de messages maxMessages sur une session ServerSession. Laissez la case décochée pour spécifier qu'une connexion consommateur Message Queue doit charger uniquement un message à la fois. Voir la section Fonctions du serveur d'application des spécifications JMS pour en savoir plus.
