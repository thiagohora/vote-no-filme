Propriétés de destination du courtier >
De base

Cet onglet affiche les informations sur une destination courtier. Vous pouvez également l'utiliser pour modifier les propriétés que vous avez définies pour une destination dans la boîte de dialogue Ajouter destination du courtier.

Nom destination Spécifie le nom de la destination physique.

Type de destination Spécifie le type de destination (file d'attente ou rubrique).

État de la destination Spécifie si la destination est active ou en attente (flux des producteurs interrompus, flux des consommateurs interrompus ou les deux)

Nombre actuel de messages Spécifie le nombre de messages non consommés se trouvant actuellement à cette destination.

Taille totale actuelle de messages (en octets) Spécifie la mémoire utilisée par les messages non consommés actuellement stockés à cette destination.

Nombre actuel de producteurs Spécifie le nombre de producteurs actuellement associés à cette destination.

Nombre actuel de consommateurs Spécifie le nombre de consommateurs actuellement associés à cette destination. Ce nombre inclut les non durables, ainsi que les durables actifs/inactifs (rubrique seulement).

Nombre actuel de consommateurs actifs Spécifie le nombre de consommateurs actifs actuellement associés à cette destination (file d'attente seulement).

Nombre actuel de consommateurs de remplacement Spécifie le nombre de consommateurs de remplacement actuellement associés à cette destination (file d'attente seulement).

Nombre max de messages Spécifie le nombre maximal de messages non consommés autorisés à cette destination.

Taille totale max des messages Spécifie la mémoire totale maximale autorisée pour les messages non consommés à cette destination. Utilisez le menu déroulant pour choisir l'unité de mesure : octets, kilo-octets ou méga-octets.

Taille max par message (en octets) Spécifie la taille maximale autorisée pour un message unique à cette destination. Utilisez le menu déroulant pour choisir l'unité de mesure : octets, kilo-octets ou méga-octets.

Nombre maximal de producteurs Spécifie le nombre maximal de producteurs autorisés pour cette destination.

Nombre maximal de consommateurs actifs S'applique uniquement aux destinations de type file d'attente. Spécifie le nombre maximal de consommateurs qui peuvent être actifs dans le cadre d'une livraison avec équilibrage de charge à partir de cette destination.

Nombre maximal de consommateurs de remplacement S'applique uniquement aux destinations de type file d'attente. Spécifie le nombre maximal de consommateurs qui peuvent remplacer les consommateurs actifs, en cas d'échec de ces derniers pendant une livraison avec équilibrage de charge à partir de cette destination.

Comportement aux limites Spécifie le mode de réponse du courtier lorsque le seuil limite de mémoire est atteint pour cette destination. Les valeurs possibles sont les suivantes :

  • FLOW_CONTROL : ralentit les producteurs
  • REMOVE_OLDEST : élimine les messages les plus anciens
  • REMOVE_LOW_PRIORITY : élimine les messages de priorité la plus faible d'après l'âge des messages (le client producteur n'est pas averti de la suppression du message)
  • REJECT_NEWEST : rejette les messages les plus récents (le client producteur reçoit une exception pour le rejet des messages permanents, mais n'est pas informé du rejet des messages non permanents

Utiliser la file d'attente des messages bloqués Spécifie l'utilisation d'une file d'attente de messages bloqués pour cette destination.

Si un message est considéré comme non livrable, il est automatiquement placé dans une file d'attente spéciale appelée file d'attente des messages bloqués. Un message placé dans cette file d'attente conserve tous ses en-têtes d'origine (y compris la destination d'origine) et un certain nombre d'informations sont ajoutées aux propriétés du message pour expliquer pourquoi il est bloqué. Un administrateur ou un développeur a la possibilité d'accéder à cette file d'attente, de supprimer un message et de déterminer pourquoi il a été placé dans la file d'attente des messages bloqués.

La file d'attente des messages bloqués est créée automatiquement par le système et porte le nom mq.sys.dmq.

Pour purger les messages d'une destination :

  1. Sélectionnez la destination dans la fenêtre de résultats de la console d'administration.
  2. Sélectionnez Actions > Purger les messages.

Pour supprimer une destination :

  1. Sélectionnez la destination dans la fenêtre de résultats de la console d'administration.
  2. Sélectionnez Modifier > Supprimer.

Pour interrompre un flux depuis\vers une destination :

  1. Sélectionnez la destination dans la fenêtre de résultats de la console d'administration.
  2. Sélectionnez Actions > Interrompre les flux des producteurs | Interrompre le flux des consommateurs | Tout interrompre.

Pour reprendre un flux de destination interrompu :

  1. Sélectionnez la destination dans la fenêtre de résultats de la console d'administration.
  2. Sélectionnez Actions > Reprendre.
