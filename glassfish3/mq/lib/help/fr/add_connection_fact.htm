Ajouter Objet Classe de connexion

Utilisez cette boîte de dialogue pour configurer un objet de classe de connexion et l'ajouter à un magasin d'objets.

Les classes de connexion sont des objets administrés utilisés par les clients JMS pour se connecter à leur courtier. Cet objet sert également à spécifier le comportement de la connexion et comment le moteur d'exécution du client l'utilise pour accéder au courtier.

Nom de recherche Utilisez ce champ pour spécifier le nom de recherche JNDI de l'objet de classe de connexion.

Type de classe Sélectionnez l'élément de menu spécifiant le type de connexion produit par la classe : ConnectionFactory, QueueConnectionFactory ou TopicConnectionFactory. Si le message doit faire partie d'une transaction distribuée, sélectionnez XAConnectionFactory, XAQueueConnectionFactory ou XATopicConnectionFactory.

Lecture seule Cochez cette case pour que l'objet de classe de connexion soit en lecture seule. Si l'objet est en lecture seule, les applications clientes ne pourront pas modifier les attributs de la classe de connexion à l'aide d'appels api.

En configurant l'objet de classe de connexion, vous spécifiez les propriétés communes à toutes les connexions produites par la classe. Vous utilisez les onglets présentés dans la boîte de dialogue pour configurer la classe de connexion. Cliquez sur l'un des liens suivants pour obtenir des informations sur un onglet particulier.

  • Cliquez sur Restaurer les paramètres par défaut pour réinitialiser tous les champs de toutes les boîtes de dialogue sur leurs valeurs par défaut.
  • Cliquez sur OK pour créer l'objet de classe de connexion.

Pour modifier les propriétés d'un objet de classe de connexion :

  1. Sélectionnez le dossier Classes de connexion dans la fenêtre de navigation de la console d'administration.
  2. Sélectionnez l'objet de classe de connexion dans la fenêtre de résultats de la console d'administration.
  3. Sélectionnez Actions > Propriétés pour afficher la boîte de dialogue Propriétés d'objet de classe de connexion.
