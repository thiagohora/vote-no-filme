Propriétés JMSX

Utilisez cet onglet pour spécifier de nouveau les propriétés définies par JMS devant être configurées par Message Queue.

Message Queue peut prendre en charge un certain nombre d'attributs définis par JMS, mais ceci est coûteux en termes de ressources. Pour préserver les ressources, vous devez identifier les propriétés réellement nécessaires à l'application client et activer ces seules propriétés en cochant la case appropriée.

Activer la propriété de message JMSXAppID : spécifie si Message Queue doit activer la propriété définie par JMS, JMSXAppID (identité de l'application émettant le message), sur les messages produits. Par défaut : faux

Activer la propriété de message JMSXUserID : spécifie si Message Queue doit activer la propriété définie par JMS, JMSXUserID (identité de l'utilisateur émettant le message), sur les messages produits. Par défaut : faux

Activer la propriété de message JMSXProducerTXID : spécifie si Message Queue doit activer la propriété définie par JMS, JMSXProducerTXID (identifiant de la transaction au sein de laquelle le message a été produit), sur les messages produits. Par défaut : faux

Activer la propriété de message JMSXConsumerTXID : spécifie si Message Queue doit activer la propriété définie par JMS, JMSXConsumerTXID (identifiant de la transaction au sein de laquelle le message a été consommé), sur les messages consommés. Par défaut : faux

Activer la propriété de message JMSXRcvTimestamp : spécifie si Message Queue doit activer la propriété définie par JMS, JMSXRcvTimestamp (l'heure où le message est livré au consommateur), sur les messages consommés. Par défaut : faux
