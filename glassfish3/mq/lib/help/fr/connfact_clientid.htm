Identification du client

Utilisez cet onglet pour spécifier à nouveau les valeurs par défaut du nom d'utilisateur et du mot de passe ainsi que le mode d'identification du client (ClientID) sur un courtier. Les valeurs ClientId servent essentiellement à suivre les abonnés durables.

Nom d'utilisateur par défaut Spécifie le nom d'utilisateur par défaut qui sera utilisé pour l'authentification par le courtier. La valeur par défaut est guest.

Mot de passe par défaut Spécifie le mot de passe par défaut qui sera utilisé pour l'authentification par le courtier. La valeur par défaut est guest.

Désactiver l'API JMS setClientID() Spécifie que l'application cliente ne peut pas utiliser les appels API pour modifier la valeur ClientID. Il est généralement préférable que ce soit vous, l'administrateur, qui contrôliez l'attribution des valeurs ClientID.

Si vous cochez cette case, vous pouvez utiliser le champ Configurer ClientID comme pour spécifier un identifiant pour ce client. Si vous ne spécifiez pas cette valeur, une valeur par défaut, égale à l'adresse IP de l'hôte client, est attribuée par le système. De ce fait, afin que ce soit plus pratique pour les développeurs, l'application cliente se voit interdire la création de plus d'une connexion : le système ne permet pas deux connexions avec le même identifiant client.

Configurer ClientID comme Spécifie un identifiant pour le client. Si vous ne spécifiez pas cette valeur et que l'appel api déterminant l'identifiant client est désactivé, une valeur par défaut, égale à l'adresse IP de l'hôte client, est attribuée par le système. De ce fait, afin que ce soit plus pratique pour les développeurs, l'application cliente se voit interdire la création de plus d'une connexion : le système ne permet pas deux connexions avec le même identifiant client.
